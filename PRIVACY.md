# Privacy Policy — epicsagas plugins

**Last updated:** 2026-04-14

## Summary

All epicsagas plugins run **locally on your machine**. We collect nothing.

## Plugins Covered

This policy covers all plugins distributed through the [epicsagas/claude-plugins](https://github.com/epicsagas/claude-plugins) marketplace:

- **epic** (epic-harness) — autonomous agent harness
- **transpile** (llm-transpile) — token-optimized document reader
- **alcove** — MCP documentation server

## Data Storage

- All data generated by these plugins is stored **locally on your filesystem**.
- epic: `~/.harness/` (SQLite database, session logs, evolved skills)
- transpile: No persistent storage. Operates in-memory on each Read call.
- alcove: `~/.alcove/` (document index, vector embeddings, search cache)
- **No data is transmitted to any external server, cloud service, or third-party API.**

## Data Collection

**We collect zero data.** Specifically:

- No personal information
- No usage statistics or telemetry
- No crash reports
- No network requests to external services

All plugin communication happens via **local stdio** (stdin/stdout) with the host application or via **localhost-only** HTTP servers when explicitly started by the user.

## Data Control

You own all data generated by these plugins. You can inspect, export, or delete it at any time by removing the relevant local directories (`~/.harness/`, `~/.alcove/`).

## Third-Party Services

These plugins use **no third-party services** at runtime. All binaries are self-contained with no external dependencies.

## Open Source

All plugins are open source under the Apache-2.0 license. You can audit the full source code:

- [epic-harness](https://github.com/epicsagas/epic-harness)
- [llm-transpile](https://github.com/epicsagas/llm-transpile)
- [alcove](https://github.com/epicsagas/alcove)

## Changes

If this policy changes, the update will be reflected in this file with an updated date.
